Описание проекта
Веб-приложение для управления королевством во время игры в авторскую настольную игру.
Пользователь регистрируется, создаёт или выбирает своё королевство, управляет ресурсами, населением, экономикой, юнитами, а также торговыми запросами. Данные сохраняются в базе SQLite. Приложение построено на FastAPI с использованием SQLAlchemy (ORM), шаблонов Jinja2, клиентская логика на чистом JavaScript с сохранением промежуточных изменений в sessionStorage/localStorage и синхронизацией с сервером через API.

Основные возможности
Регистрация / авторизация пользователей (пароли хранятся в открытом виде – для учебного проекта).

Создание нескольких королевств на одного пользователя (уникальные имена в пределах пользователя).

Главная панель управления королевством:

Ресурсы: пшеница, золото, железо, фабрика, инструменты, оружие.

Население: количество людей, уровень счастья, уровень института (образование), уровень развития.

Экономика: базовая прибыль, казна, дополнительные доходы / расходы, кнопка «Обновить ход» (пересчитывает казну).

Юниты: 11 типов войск и построек.

Торговля и запросы: добавление заявок на торговлю с другими королевствами (имя королевства, объект, количество). Можно удалять заявки кликом.

Загрузка и сохранение герба королевства (изображение хранится в localStorage, в базе не сохраняется).

Автосохранение всех полей (в localStorage + синхронизация с сервером по кнопке «Сохранить»).

Возможность переключения между королевствами из файлового менеджера.

Установка и запуск
Клонируйте репозиторий (или создайте проект с указанной структурой файлов).

Установите зависимости
Желательно использовать виртуальное окружение:

bash
pip install fastapi uvicorn sqlalchemy pydantic python-multipart jinja2
Запустите приложение
Из корневой папки проекта выполните:

bash
uvicorn main:app --host 127.0.0.1 --port 8000 --reload
--reload автоматически перезагружает сервер при изменениях кода (удобно для разработки).

Можно указать свой --host и --port.

Откройте браузер и перейдите по адресу:
http://127.0.0.1:8000

API эндпоинты
Метод	Endpoint	Описание
GET	/	Страница входа
GET	/reg	Страница регистрации
POST	/reg	Обработка регистрации (перенаправление) – дублирует /file_maneger/reg
POST	/login	Вход в систему (сессия на основе user_id)
GET	/file_maneger	Список королевств текущего пользователя
POST	/file_maneger/reg	Регистрация нового пользователя + автоматический вход
GET	/kingdom	Страница управления королевством (параметр ?kingdom_id=...)
POST	/kingdom/save	Сохранение данных королевства (JSON) – создание нового или обновление
Примечания по реализации
Хранение паролей: для простоты учебного проекта пароли не хэшируются. В реальном приложении используйте passlib или bcrypt.

Сохранение герба: изображение хранится только в localStorage браузера. При смене устройства или очистке хранилища герб теряется.

Запросы (торговля): Вы можете добавлять и удалять заявки. Каждая заявка содержит имя королевства-партнёра, объект торговли и количество. При сохранении королевства заявки синхронизируются с БД.

Автосохранение: все поля (кроме герба) дублируются в localStorage при каждом изменении. Кнопка «Сохранить» отправляет полное состояние на сервер и обновляет БД.

Начальные значения заданы константами в main.py:

Ресурсы: пшеница=30, золото=1, железо=0, фабрика=20, инструменты=0, оружие=0.

Население: люди=17, счастье=100, институт=1, уровень=1.

Экономика: прибыль=10, казна=0, плюс=0, минус=7.
