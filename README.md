Описание проекта
Веб-приложение для управления королевством во время игры в авторскую настольную игру.
Пользователь регистрируется, создаёт/выбирает своё королевство, управляет ресурсами, населением, экономикой, юнитами и торговыми запросами.
Данные сохраняются в SQLite, пароли хешируются, герб королевства хранится в БД (base64).
Интерфейс адаптирован для мобильных устройств.

Основные возможности
Регистрация / авторизация с хешированием паролей (bcrypt).

Несколько королевств на одного пользователя (уникальные имена в пределах пользователя).

Панель управления королевством:

Ресурсы: пшеница, золото, железо, фабрика, инструменты, оружие.

Население: люди, счастье, институт (образование), уровень развития.

Экономика: прибыль, казна, дополнительные доходы/расходы, кнопка «Обновить ход» (пересчитывает казну с учётом множителей уровня).

Юниты: 11 типов войск и построек.

Торговые запросы: добавление / удаление заявок (имя королевства, объект, количество).

Герб королевства: загрузка изображения, сохранение в БД как base64.

Страница профиля:

количество королевств, лучшее королевство (по уровню, затем по золоту), дата последнего входа.

Мобильная адаптация: сетка Bootstrap, увеличенные области касания, удобное отображение на узких экранах.

Файловый менеджер для выбора и удаления королевств.

Установка и запуск
1. Клонируйте репозиторий / создайте проект с указанной структурой
2. Установите зависимости
Рекомендуется использовать виртуальное окружение:

pip install fastapi uvicorn sqlalchemy pydantic python-multipart jinja2 bcrypt
3. Запустите приложение
Из корневой папки проекта выполните:

uvicorn main:app --reload
--reload автоматически перезагружает сервер при изменениях кода (удобно для разработки).

При необходимости укажите другой хост/порт: --host 0.0.0.0 --port 8000

4. Откройте браузер
Перейдите по адресу: http://127.0.0.1:8000

Важно: если вы ранее запускали старую версию, удалите database.db перед первым запуском, чтобы таблицы пересоздались с полем avatar и хешированием паролей.

Особенности реализации
Безопасность
Пароли хранятся в хешированном виде (bcrypt).

Сессии реализованы через SessionMiddleware (секретный ключ в коде).

Герб королевства
Изображение загружается через FileReader и преобразуется в base64.

Base64-строка сохраняется в поле avatar таблицы kingdoms (SQLite, тип Text).

При загрузке королевства аватар извлекается из БД и отображается.

Никакие файлы на диске не хранятся – всё в БД (просто для учебного проекта; для реального использования лучше хранить файлы и пути к ним).

Экономика
Прибыль рассчитывается по формуле:
(люди × золото) + сумма(доходы) - сумма(расходы), где расходы умножаются на коэффициент уровня (1..4).

Кнопка «Обновить ход» добавляет рассчитанную прибыль к казне.

Поля eco_plus и eco_minus поддерживают арифметические выражения (например, 5+3 или 10-2).

Торговые запросы
Добавляются через форму (три слова: имя объект количество).

Отображаются в виде списка с кнопкой удаления.

Клик по запросу заполняет поля «Объект» и «Королевство» в блоке торговли.

Запросы сохраняются в связанной таблице requests.

Мобильная адаптация
Мета-тег viewport с user-scalable=yes.

Сетка Bootstrap с классами col-6 col-md-2 и т.п.

Кнопки занимают всю ширину на малых экранах (flex-grow-1).

Поля ввода имеют размер шрифта 16px (предотвращает масштабирование на iOS).

Список запросов ограничен по высоте и прокручивается (max-height: 200px; overflow-y: auto).

Значения по умолчанию
При создании нового королевства HTML-атрибуты value устанавливают начальные данные (30 пшеницы, 1 золото, 17 людей и т.д.).

JavaScript при необходимости дополняет их (например, для аватара).

Возможные доработки
Сохранение герба на диск (вместо base64 в БД) для снижения нагрузки на базу.

Автосохранение в localStorage с восстановлением при потере сессии.

Валидация на сервере (сейчас минимальная).

Поддержка нескольких языков.

История действий (лог изменений).

Примечания для разработчика
После добавления новой колонки в модель обязательно удалите database.db, иначе SQLite не применит изменения.

Для отладки аватара в консоли сервера выводятся сообщения о получении base64.

Все CSS-файлы объединены: custom.css содержит основную тему и мобильные стили, style.css – только мобильные медиазапросы (можно объединить в один).

Проект не требует установки дополнительных сервисов (БД – SQLite).
